Приоритетная очередь – 2
Ограничение по времени на тест 3 секунды
Ограничение по памяти на тест 256 мегабайт

Реализуйте приоритетную очередь. Ваша очередь должна поддерживать следующие операции: добавить элемент, извлечь минимальный элемент,
уменьшить элемент, добавленный во время одной из операций.
Если какой-нибудь decrease-key уменьшает уже удаленный элемент, то ничего делать не нужно.
Все операции нумеруются по порядку, начиная с 1.

Входные данные
Содержится описание операций со очередью. В очередь помещаются и извлекаются только целые числа, не превышающие 10^9 по абсолютной величине.
Гарантируется, что для любого decrease-key x v из входных данных операция под номером x является push.

Выходные данные
Выведите последовательно результат выполнения всех операций extract-min из двух целых чисел: значение элемента и номер операции
push, при котором этот элемент был добавлен. Если в очереди есть несколько минимальных элементов, выведите любой. Если перед очередной
операцией extract-min очередь пуста, выведите звездочку.

Пример
Входные данные

push 3
push 4
push 2
extract-min
decrease-key 2 1
extract-min
extract-min
extract-min

Выходные данные

2 3
1 2
3 1
*
